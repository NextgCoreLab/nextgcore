Why NextGCore?

An agentic core network that thinks, adapts, and secures itself.

🤖

AI-Native Intelligence

NWDAF with 4-layer analytics engine (real-time anomaly detection, predictive load forecasting, prescriptive handover optimization, autonomous actions). ML model lifecycle management with ONNX inference.

🎯

Intent-Based Automation

Declarative intent system — express network goals ("optimize slice latency below 10ms") and let the agentic core figure out the actions. Conflict detection, priority resolution, and RL-based learning from outcomes.

🛡

Zero-Trust Security

SEPP with PRINS encryption, per-PLMN trust scoring, anomaly-based blocking, and rate limiting. OAuth2 JWT with cryptographic signing. TLS on all SBI interfaces.

☁

Cloud Native

Kubernetes with 16 pods, Helm charts, Prometheus metrics on every NF, Jaeger distributed tracing, horizontal scaling. Docker Compose for dev, Kubernetes for production.

🚀

Edge Intelligence

EES manages edge application servers with proximity-based discovery, capability matching, and UE context transfer for seamless edge relocation. 3-tier compute: edge → regional → cloud.

⚖

3GPP Rel-15 to Rel-20

Full 5G SA, 5G-Advanced (Rel-17/18), and 6G research (Rel-20). MBS multicast, PIN IoT networks, NSACF slice admission, LMF positioning with 10+ methods, and beyond.

Trust Scoring

Dynamic trust levels (Unknown → Initial → Established → High) per remote PLMN. Anomaly scoring 0-100 with threshold blocking.

PRINS Encryption

JSON patch-based selective IE protection — encrypt SUPI, PEI, GPSI while leaving routing headers readable. Data-type profiles per IE.

Rate Limiting

Per-PLMN RPS limits, integrity/confidentiality requirements, strict vs. permissive policies. Real-time policy evaluation on every request.

Key Workflows

End-to-end procedures from registration to AI-driven optimization.

1. UE Registration & Authentication (TS 23.502 §4.2.2)

Full 5G SA initial registration with SUCI concealment, 5G-AKA mutual authentication, NAS security activation, and policy retrieval.

UE→NAS: Registration Request (SUCI = ECIES-encrypted SUPI, Requested NSSAI, UE capabilities)→gNB
gNB→NGAP: InitialUEMessage (NAS-PDU, User Location Info, RRC Establishment Cause)→AMF
AMF→Nnssf_NSSelection (Requested NSSAI, TAI) → returns Allowed NSSAI, target AMF set→NSSF
AMF→Nausf_UEAuthentication (SUCI, serving network name)→AUSF
AUSF→Nudm_UEAuthentication_Get (SUCI) → SUPI resolution + auth vectors (RAND, AUTN, HXRES*, K_AUSF)→UDM
AMF→NAS: AuthenticationRequest (RAND, AUTN) → UE computes RES*→UE
UE→NAS: AuthenticationResponse (RES*) → AMF forwards to AUSF for HXRES* verification→AMF
AMF→NAS: SecurityModeCommand (selected NAS ciphering: NEA1/2, integrity: NIA1/2, K_AMF derived)→UE
UE→NAS: SecurityModeComplete (encrypted + integrity-protected from this point on)→AMF
AMF→NGAP: InitialContextSetupRequest (K_gNB security key, Allowed NSSAI, UE AMBR, DRB config)→gNB
AMF→Nudm_UECM_Registration (SUPI, AMF instance ID) + Nudm_SDM_Get (subscriber data, AM policies)→UDM
AMF→Npcf_AMPolicyControl_Create (SUPI, Access Type, Allowed NSSAI) → AM policy decisions→PCF
AMF→NAS: Registration Accept (5G-GUTI, Allowed NSSAI, LADN info, configured NSSAI)→UE
UE→NAS: Registration Complete — UE is now CM-CONNECTED, RM-REGISTERED→AMF

2. PDU Session Establishment (TS 23.502 §4.3.2)

IPv4 PDU session with policy-controlled QoS, PFCP forwarding rules, GTP-U tunnel setup, and bitmap-based IP allocation.

UE→NAS: PDU Session Establishment Request (PDU Session ID, S-NSSAI, DNN="internet", PDU type=IPv4)→AMF
AMF→SMF selection (S-NSSAI, DNN) via NRF discovery → Nsmf_PDUSession_CreateSMContext→SMF
SMF→Nudm_SDM_Get (session management subscription data for DNN + S-NSSAI)→UDM
SMF→Npcf_SMPolicyControl_Create → PCC rules, QoS decisions (QFI, AMBR, ARP, 5QI), charging rules→PCF
SMF→IP allocation: bitmap pool 10.45.0.0/16 → allocates 10.45.0.3 (first free bit)→Pool
SMF→UPF selection (DNN, TAI, load) → PFCP Session Establishment (PDR: match SDF 5-tuple, FAR: forward GTP-U, QER: enforce AMBR, URR: volume reporting)→UPF
UPF→PFCP Session Establishment Response (F-TEID for uplink = 10.11.0.7:gtp-u, DL-TEID assigned)→SMF
SMF→Namf_Communication_N1N2MessageTransfer (NAS: PDU Session Accept + N2: GTP tunnel info + QoS flows)→AMF
AMF→NGAP: PDUSessionResourceSetupRequest (GTP-U TEID, QoS flow list, NAS PDU)→gNB
gNB→RRC Reconfiguration (DRB setup, QoS mapping) + forwards NAS to UE→UE
UE→PDU Session active: IP=10.45.0.3, DNN=internet, QFI=1, GTP tunnel established→Data Ready

3. Xn Handover (TS 23.502 §4.9.1)

Intra-AMF handover with path switch, GTP tunnel update, and seamless session continuity.

UE→RRC: MeasurementReport (neighbor cell RSRP > serving + hysteresis)→Source gNB
Source gNB→Xn: HandoverRequest (UE context, PDU sessions, security keys, target cell ID)→Target gNB
Target gNB→Xn: HandoverRequest Acknowledge (DL forwarding tunnel, target-to-source container)→Source gNB
Source gNB→RRC: RRCReconfiguration (target cell config, new C-RNTI)→UE
UE→Detach from source → Random access on target → RRC Reconfiguration Complete→Target gNB
Target gNB→NGAP: PathSwitchRequest (new GTP-U TEID, security key, PDU session list)→AMF
AMF→Nsmf_PDUSession_UpdateSMContext (new AN tunnel info) → SMF sends PFCP Modification to UPF→SMF → UPF
AMF→NGAP: PathSwitchRequest Acknowledge — handover complete, data flows via new path→Target gNB

4. NWDAF AI-Driven Optimization (TS 23.288)

4-layer analytics pipeline: real-time anomaly detection → ONNX prediction → prescriptive recommendation → autonomous action.

AMF/SMF→Nnwdaf_EventsSubscription (analytics_id: NF_LOAD, target_period: 5min, consumer: PCF)→NWDAF
gNBs→Data collection: UE measurements (RSRP, RSRQ, SINR), cell load (PRB usage, UE count, throughput)→NWDAF
NWDAF L1→Real-time: Z-score anomaly detection on RSRP/SINR → anomaly score 78/100 detected on cell 0x10→Alert
NWDAF L2→Predictive: ONNX trajectory model → UE moving NE at 30km/h, will leave cell in ~45s, time-to-overload: 2min→Forecast
NWDAF L3→Prescriptive: Analytics payload (UE_MOBILITY, NF_LOAD, ABNORMAL_BEHAVIOR) → handover recommendation type: LOAD_BALANCING→Decision
NWDAF L4→Autonomous action: AdjustHandoverParams (cell 0x10 → lower HO threshold by 3dB) + TriggerLoadBalancing→AMF
NWDAF→Nnwdaf_AnalyticsInfo notification → push analytics to all subscribed consumers (PCF, AMF)→Consumers
NWDAF→ML model update: record outcome (handover success/fail) → retrain trajectory model → v3.1 deployed→Model Registry

5. Slice Admission & Quota Control (TS 23.502 §4.2.9)

End-to-end slice selection with NSSF, quota enforcement via NSACF, and policy binding.

UE→Registration Request (Requested NSSAI: SST=1/SD=0x010203 eMBB, SST=2 URLLC)→AMF
AMF→Nnssf_NSSelection (Requested NSSAI, TAI, PLMN) → Allowed NSSAI, AMF set for each slice→NSSF
AMF→Nnsacf_UEAdmission (SUPI, S-NSSAI=SST:1) → check per-slice quota→NSACF
NSACF→Quota check: SST=1 → 150 UEs / 200 max, 89 PDU sessions / 500 max → ADMITTED, counters incremented→AMF
AMF→Nnsacf_UEAdmission (SUPI, S-NSSAI=SST:2) → SST=2: 200 UEs / 200 max → REJECTED (QUOTA_EXCEEDED)→NSACF
AMF→Registration Accept: Allowed NSSAI = [SST=1/SD=0x010203], Rejected NSSAI = [SST=2, cause: QUOTA]→UE
AMF→On UE deregistration: Nnsacf_UERelease → counters decremented, slot freed for next UE→NSACF

6. Edge Application Relocation (TS 23.558)

Seamless MEC edge relocation with EAS discovery, context preservation, and traffic re-routing.

UE→Mobility event: TAC changes from 0x01 to 0x02 (detected by AMF via Tracking Area Update)→AMF
AMF→Location change notification → EES evaluates if current EAS is still optimal→EES
EES→POST /nees-easdiscovery/v1/discover (app_id="ar-nav", TAC=0x02) → ranked EAS list by distance score→Discovery
EES→Best match: EAS "edge-east-2" (type: AR, compute: 80%, distance_score: 0.92, TAC: 0x02)→Selected
EES→GET /nees-uecontexttransfer/v1/contexts/{supi} → fetch UE app state from old EAS (session data, model weights, UI state)→Old EAS
EES→PATCH /nees-uecontexttransfer/v1/contexts/{supi} → transfer context to new EAS→New EAS
New EAS→Context restored, app resumes — UE sees no interruption. SMF updates UL CL / ULCL rules→UE

7. Inter-PLMN Roaming Security (TS 33.501 / TS 29.573)

N32 security with handshake, PRINS selective encryption, zero-trust policy evaluation, and anomaly detection.

Home SEPP→N32c: SecurityCapabilityExchange (supported ciphers, data-type profiles, PRINS version)→Visited SEPP
Visited SEPP→N32c: SecurityCapabilityResponse → mutual cipher selection, shared secret derivation→Home SEPP
Home NF→SBI: Nudm_SDM request for roaming subscriber (contains SUPI, PEI, GPSI in JSON body)→Home SEPP
SEPP→Zero-trust policy: trust_level=Established (85/100), anomaly_score=12, RPS=45/100 → PASS→Policy Engine
SEPP→PRINS: Apply data-type profile → encrypt SUPI, PEI, GPSI IEs (JSON patch), integrity-protect message→Protected
Home SEPP→N32f: Forward protected message (routing headers readable, subscriber IEs encrypted)→Visited SEPP
Visited SEPP→Verify integrity → Decrypt PRINS IEs → Zero-trust check on incoming → Route to destination NF→Visited NF
SEPP→Audit log: timestamp, source_plmn, dest_plmn, trust_score, anomaly_score, action=FORWARD→Audit Trail

8. UE Positioning via LMF (TS 23.273)

Multi-method positioning with NRPPa measurement collection, coordinate calculation, and QoS-based accuracy.

App / AMF→POST /nlmf-location/v1/determine-location (SUPI, QoS=HIGH_ACCURACY, methods=[DL-TDOA, Multi-RTT])→LMF
LMF→POST /nlmf-location/v1/measurements (methods, cells=[0x10, 0x11, 0x12], measurement_period=500ms)→gNBs
gNBs→NRPPa reports: RSRP=-85dBm, timing_advance=12.3μs, AoA=45.2°, RTT=234ns per cell→LMF
LMF→Multi-RTT trilateration from 3 cells → lat: 41.0082, lon: 28.9784, alt: 35m, accuracy: ±2.5m→Position
LMF→PUT /nlmf-location/v1/ue-locations/{supi} → store result + return to requesting entity→App / AMF

9. Multicast/Broadcast Session (TS 23.247)

TMGI-based group session creation, UE join/leave, N4mb PFCP setup with UPF for multicast delivery.

AF→POST /nmbsmf-mbssession/v1/mbs-sessions (type: MULTICAST, tmgi, service_area, qos: QCI=7)→MBSMF
MBSMF→Session created: id=mbs-001, TMGI=09f107fffe, state=CREATED→Ready
UE-1→POST /mbs-sessions/mbs-001/members (supi=imsi-999700000000001) → JOINED (member_count: 1)→MBSMF
UE-2→POST /mbs-sessions/mbs-001/members (supi=imsi-999700000000002) → JOINED (member_count: 2)→MBSMF
AF→POST /mbs-sessions/mbs-001/activate → MBSMF establishes N4mb PFCP session with UPF→MBSMF
MBSMF→N4mb: PFCP Session Establish (multicast PDR, FAR with DL-TEID for each gNB, mcast group IP)→UPF
UPF→Multicast forwarding active: DL data → replicated to all member gNBs via GTP-U tunnels→gNBs → UEs

Quick Start

Get the full agentic 6G core running in minutes.

Build from Source

# Clone the repository git clone https://github.com/NextgCoreLab/nextgcore.git cd nextgcore/src # Build all 22 NFs (release mode) cargo build --release # Binaries are in target/release/ ls target/release/nextgcore-*

Docker Compose

# Start all core services + MongoDB cd docker/rust docker compose up -d # Check all services are running docker compose ps # Run E2E tests bash e2e-test.sh

Kubernetes (Production)

# Deploy to K8s cluster (16 pods) cd k8s && bash deploy.sh # Verify all pods running kubectl get pods -n nextgcore # Run full E2E suite (70 tests) bash e2e-test.sh # Check Prometheus metrics curl http://amf:9090/metrics | head # View traces in Jaeger kubectl port-forward svc/jaeger 16686:16686
